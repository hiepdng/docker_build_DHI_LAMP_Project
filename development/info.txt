HTTPD CA Certificates:

# 1. Copy your custom .crt file into the Debian local CA directory
COPY my-internal-ca.crt /usr/local/share/ca-certificates/my-internal-ca.crt
# 2. Update the certificate store to include your custom CA
RUN apt-get update && \
    apt-get install -y ca-certificates && \
    update-ca-certificates
# 3. Ensure your app knows where the updated bundle lives
ENV SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt

Behind the scenes on Debian, /etc/ssl/certs/ca-certificates.crt is generated by the update-ca-certificates utility.
That utility reads certificate files from /usr/share/ca-certificates/ and /usr/local/share/ca-certificates/, creates 
individually hashed symlinks in /etc/ssl/certs/, and then concatenates them all into the single ca-certificates.crt file
for applications that require a single bundle

-In httpd.conf, use SSLCACertificateFile directive

-How to create my-internal-ca.crt
# openssl genrsa -out my-internal-ca.key 2048
# openssl req -x509 -new -nodes -key my-internal-ca.key -sha256 -days 1826 -out my-internal-ca.crt  -subj '/C=US/ST=state/L=city/O=example/OU=sale/CN=server.example.com/emailAddress=you@example.com'
